## 4. ERD

초기 초안은 Mermaid를 사용합니다. 이후 필요하면 데이터베이스별 스키마로 확장합니다.

```mermaid
erDiagram
  USER ||--o{ ITEM : owns
  USER {
    string id
    string email
    datetime created_at
  }
  ITEM {
    string id
    string user_id
    string status
    datetime created_at
  }
```

### 선택적 DBML

ERD가 너무 크거나 데이터베이스별 세부사항이 중요해지면 DBML을 보조 산출물로 사용합니다.

```dbml
Table users {
  id varchar [pk]
  email varchar [unique, not null]
  created_at timestamp
}

Table items {
  id varchar [pk]
  user_id varchar [not null, ref: > users.id]
  status varchar
  created_at timestamp
}
```

## 12. 초안 완료 체크리스트

- 핵심 엔티티가 나열되어 있고 기능과 연결되어 있다.
- ERD가 주요 관계를 보여준다.

- 소유권과 접근 규칙이 정의되어 있다.
- 중요한 조회를 위한 조회 패턴 또는 인덱스가 기록되어 있다.

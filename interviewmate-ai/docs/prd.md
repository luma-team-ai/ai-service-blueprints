# InterviewMate AI PRD

## 1. 문서 목적

이 문서는 신입 개발자 취업 준비생을 위한 한국어 AI 취업 준비 플랫폼, InterviewMate AI의 제품 요구사항과 의사결정 기준선을 정의합니다.

현재 단계는 기획과 프로토타입 준비입니다. 향후 구현은 저장 계정, 문서 업로드, 음성/영상 면접 연습, 모바일 클라이언트로 확장하기 전에 집중된 웹앱 프로토타입부터 시작합니다.

## 2. 프로젝트 개요

InterviewMate AI는 주니어 개발자 지원자가 하나의 작업 공간에서 입사 지원을 준비하도록 돕습니다. 사용자는 한국어 자기소개서를 생성하고 개선하며, 이력서를 점검하고, 어울리는 직무나 회사 유형을 탐색하고, 면접을 연습하고, 우선순위가 있는 다음 행동을 받을 수 있어야 합니다.

빠른 프로토타이핑에는 Google AI Studio를 사용할 수 있지만, 장기적으로는 일반적인 저장소 기반 개발로 전환할 수 있어야 합니다. 제품은 저장된 사용자 이력, 문서 업로드, GitHub 또는 포트폴리오 분석, 회사별 면접 연습, 음성/영상 피드백으로 확장될 수 있습니다.

## 3. 프로젝트 목표

사용자는 자주 다음 문제를 겪습니다.

- 좋은 한국어 자기소개서를 쓰기 어렵다.
- 자신의 이력서가 경쟁력 있는지 판단하기 어렵다.
- 프로젝트, 포트폴리오, 부트캠프 경험, 학교 프로젝트, GitHub 활동을 채용 맥락의 근거로 바꾸기 어렵다.
- 현재 프로필에 맞는 개발 직무나 회사 유형을 결정하기 어렵다.
- 실제 면접에 가까운 환경에서 연습하기 어렵다.

InterviewMate AI는 다음을 제공해야 합니다.

- 사용자가 제공한 경험을 구체적인 입사 지원 자료로 바꾼다.
- 이유, 예시, 우선순위가 있는 이력서 피드백을 제공한다.
- 적합 근거와 부족한 준비 항목을 포함해 직무 또는 회사 방향을 추천한다.
- 현실적인 면접 질문과 답변 피드백을 생성한다.
- 사용자가 수정과 최종 주장에 대한 통제권을 유지하게 한다.

## 4. 타깃 사용자

주요 사용자:

- 한국어로 취업을 준비하는 신입 개발자 지원자.
- 학생, 부트캠프 수료생, 졸업 예정자, 최근 졸업자, 커리어 초년생.
- 프로젝트, 포트폴리오, GitHub 링크, 인턴십, 학교 활동, 기술 학습 이력은 있으나 정규직 실무 경험은 제한적인 사용자.

초기 집중 사용자:

- 한국어 입사 지원을 준비하는 주니어 개발자 지원자.
- 프로젝트나 이력서 재료는 있지만 채용 맥락에 맞게 정리하는 데 도움이 필요한 사용자.
- 넓은 격려보다 실질적인 다음 행동을 원하는 사용자.

## 5. 핵심 가치 제안

- 사용자의 실제 경험에 근거한 한국어 자기소개서 초안을 생성한다.
- 이력서의 구조, 근거, 직무 적합성, 명확성을 점검한다.
- 확인할 수 없는 회사 데이터를 아는 척하지 않고 적합한 직무 또는 회사 유형을 추천한다.
- 면접 연습과 답변 피드백을 제공한다.
- 강점, 리스크, 다음 행동이 보이는 피드백 리포트를 만든다.
- 사용자를 대신해 조용히 모두 고쳐 쓰기보다 제안, 설명, 비교를 통해 사용자 주도권을 유지한다.

## 6. MVP 범위

### 6.1 포함

- 취업 준비 작업을 시작하는 대시보드.
- 사용자 프로필과 자료 입력: 목표 직무, 기술 스택, 프로젝트, 포트폴리오 링크, GitHub 링크, 기존 이력서, 채용공고, 회사 선호 조건.
- 자기소개서 생성과 편집.
- 전후 비교 예시를 포함한 이력서 첨삭.
- 프로필, 이력서, 포트폴리오, 채용공고 입력에 기반한 회사 또는 직무 매칭.
- 가벼운 텍스트 기반 면접 연습.
- 우선순위가 있는 다음 행동을 담은 피드백 리포트.
- 저장 계정 없이도 프로토타입을 테스트할 수 있는 샘플 데이터.

### 6.2 제외

- 합격 보장 또는 합격 확률 단정.
- 성과, 자격, 수치, 프로젝트, 고용 이력, 면접 성과의 조작.
- 네이티브 모바일 앱.
- 실시간 음성 또는 영상 면접.
- 표정 또는 음성 분석.
- 결제와 구독 구현.
- 명시적으로 요청되지 않은 외부 채용 사이트, GitHub, 캘린더, 저장소 연동.

## 7. 다음 범위와 향후 범위

다음 범위:

- 구조화된 모의면접 세션.
- 회사별 면접 질문 생성.
- 면접 답변 이력과 피드백.
- 사용자 계정, 저장된 작업, 리포트 이력.
- 이력서와 포트폴리오 문서 업로드.

향후 범위:

- 음성 기반 모의면접.
- 영상 기반 모의면접.
- 표정 피드백.
- 목소리 톤, 속도, 필러 표현, 자신감 피드백.
- 모바일 앱 클라이언트.
- 채용 사이트, 캘린더, 저장소, GitHub, 포트폴리오 서비스 연동.

## 8. 사용자 흐름

### 자기소개서 생성

1. 사용자가 목표 직무, 강점, 프로젝트 경험, 선택적으로 채용공고를 입력한다.
2. 시스템이 관련 경험 신호를 추출한다.
3. AI가 한국어 자기소개서 초안을 생성한다.
4. AI가 각 문단을 뒷받침하는 근거를 설명한다.
5. 사용자가 편집, 저장, 톤 변경 요청을 할 수 있다.

### 이력서 개선

1. 사용자가 이력서 내용을 업로드하거나 직접 입력한다.
2. 사용자가 선택적으로 목표 직무, 채용공고, 포트폴리오, GitHub, 경력 메모를 추가한다.
3. 시스템이 구조화된 프로필 데이터를 추출한다.
4. AI가 부족한 근거, 약한 표현, 누락된 경험, 직무 적합성 문제를 찾는다.
5. 사용자는 수정 제안과 피드백 리포트를 받는다.
6. 사용자는 제안을 수락, 거절, 수정할 수 있다.

### 회사와 직무 매칭

1. 사용자가 이력서, 포트폴리오, 관심사, 기술 스택, 선호 조건을 제공한다.
2. 시스템이 강점, 경험 신호, 제약을 추출한다.
3. AI가 적합한 개발 직무 또는 회사 유형을 추천한다.
4. 추천에는 적합 근거, 부족한 준비, 다음 행동이 포함된다.

### 면접 연습

1. 사용자가 목표 직무, 회사 유형, 면접 난이도를 선택한다.
2. AI가 직무 관련 면접 질문을 생성한다.
3. 프로토타입에서는 텍스트로 답변하고, 향후 버전에서는 음성/영상으로 확장한다.
4. AI가 답변 구조, 기술적 명확성, 근거, 자신감 신호에 대해 피드백한다.

## 9. 권장 구현 방향

Google AI Studio 프로토타입:

- Build Mode로 풀스택 웹앱을 만든다.
- 프론트엔드는 React를 사용한다.
- Gemini API 호출은 서버사이드 Node.js 런타임에서 처리한다.
- `GEMINI_API_KEY`는 서버 시크릿에만 둔다.
- 영속 계정과 저장된 리포트가 프로토타입에 필요할 때만 Firebase Authentication과 Firestore를 사용한다.
- AI 분석, 매칭, 리포트에는 구조화된 JSON 출력을 사용한다.
- UX 문구는 한국어로 작성한다.

저장소 기반 제품 개발:

- TypeScript 우선 스택을 선호한다.
- 프론트엔드: Next.js, React, TypeScript.
- 스타일링: Tailwind CSS 또는 CSS 변수 기반의 작은 디자인 시스템.
- 백엔드: MVP는 Next.js route handlers로 시작하고, 이후 별도 API 서버로 분리할 수 있는 경로를 유지한다.
- 데이터베이스: 제품 단계에 따라 PostgreSQL, Supabase, Firebase 중 선택한다.
- AI 오케스트레이션: 타입이 있는 입력/출력을 가진 독립 서비스 모듈로 분리한다.

## 10. 제품 원칙

- 실질적이고 구체적이어야 한다. 사용자는 막연한 격려보다 구체적인 수정이 필요하다.
- 주니어 개발자에게 최적화한다.
- 사용자 주도권을 보존한다.
- 경험을 조작하지 않는다.
- 피드백은 행동 가능해야 한다. 모든 리포트에는 우선순위, 이유, 다음 행동이 있어야 한다.
- 한국어 품질이 중요하다.
- 사용자 문서, 녹음, 면접 영상은 민감 데이터로 취급한다.

## 11. AI 출력 규칙

AI 생성 피드백은 다음을 지켜야 합니다.

- 사실과 제안을 구분한다.
- 경험, 자격, 수상, 고용 이력, 측정 가능한 성과, 면접 성과를 지어내지 않는다.
- 강한 재작성에 근거가 부족하면 추가 정보를 요청한다.
- 개발자 입사 지원에 적합한 명확한 한국어를 사용한다.
- 긴 일반론보다 짧고 구체적인 예시를 선호한다.
- 사용자 글을 수정할 때는 전후 비교를 제공한다.
- 필요하면 확신 수준 또는 근거 수준을 포함한다.

AI 생성 리포트는 보통 다음을 포함합니다.

- 종합 평가.
- 주요 강점.
- 주요 리스크 또는 약점.
- 직무 또는 회사 적합성 분석.
- 추천 수정안.
- 우선순위가 있는 행동 항목.
- 선택적 재작성 예시.

## 13. 개인정보와 안전

- 문서, 오디오, 영상 분석에는 명시적 사용자 동의를 요구한다.
- 업로드 문서와 녹화/녹음 데이터 보존 정책을 저장 전에 정의한다.
- 영속 저장이 있을 경우 민감 사용자 데이터의 삭제/내보내기를 지원한다.
- 저장 파일과 민감 필드는 필요에 따라 암호화한다.
- 개인 콘텐츠 로깅을 최소화한다.
- 원본 업로드와 추출된 프로필 데이터를 분리한다.
- 자격이나 경험을 조작하지 못하도록 가드레일을 둔다.
- 추천과 피드백이 취업 결과를 보장하지 않는다고 명확히 알린다.

## 14. UX 방향

앱은 마케팅 랜딩 페이지가 아니라 취업 준비를 위한 집중된 생산성 도구처럼 느껴져야 합니다.

- 첫 화면은 자기소개서, 이력서, 매칭, 면접 연습 같은 구체 작업으로 사용자를 안내한다.
- 내비게이션은 대시보드, 자기소개서, 이력서, 매칭, 면접, 리포트 중심으로 단순하게 유지한다.
- 문서 리뷰 화면은 조밀하지만 읽기 쉽게 구성한다.
- AI 제안은 가능하면 관련 텍스트 옆에 보여준다.
- 수락, 거절, 수정 흐름을 명확하게 만든다.
- 리포트 페이지는 우선순위와 구체적인 다음 행동이 빠르게 보이게 한다.
- 면접 기능은 평가받는 느낌보다 차분하게 연습하는 느낌을 준다.

## 15. 우선순위

### P0

- 대시보드.
- 사용자 자료 입력.
- 자기소개서 생성.
- 이력서 첨삭.
- 회사 또는 직무 매칭.
- 텍스트 기반 면접 연습.
- 피드백 리포트.
- 안전한 AI 출력 동작.

### P1

- 저장된 사용자 이력.
- 문서 업로드.
- 회사별 면접 질문.
- 구조화된 채점 루브릭.
- 리포트 버전 관리.

### P2

- GitHub 또는 포트폴리오 분석.
- 채용공고 수집.
- 테스트 면접 세션.
- 멤버십 또는 접근 제어 기획.

### Future

- 음성/영상 면접.
- 표정과 음성 피드백.
- 모바일 앱.
- 결제와 구독.
- 외부 연동.

## 16. 주요 리스크

- AI가 사용자 경험을 과장하거나 근거 없는 주장을 만들어낼 수 있다.
- 이력서와 면접 데이터는 민감하므로 보존과 로깅 정책이 중요하다.
- 넓은 all-in-one 범위가 MVP를 흐릴 수 있다.
- 한국어 채용 맥락의 문구 품질이 낮으면 신뢰도가 떨어진다.
- 음성/영상 분석은 개인정보 부담이 커서 가볍게 도입하면 안 된다.
- 추천 근거가 명확하지 않으면 매칭 추천이 임의적으로 느껴질 수 있다.

## 17. 열린 질문

- 첫 구현 목표를 Google AI Studio 전용으로 할지, 저장소 기반 Next.js 프로토타입으로 할지 결정한다.

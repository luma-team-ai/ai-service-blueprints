# Google AI Studio 첫 프롬프트

## 문서 목적

이 문서는 Google AI Studio Build Mode에서 첫 프로토타입을 만들 때 사용할 시작 프롬프트를 정리한다. 목표는 팀의 프로젝트 방향과 기존 제품 문서를 모두 충족하는 한국어 웹앱 MVP를 빠르게 생성하는 것이다.

첫 프롬프트는 너무 많은 기능을 한 번에 완성하게 만들기보다, 전체 제품 구조를 보이게 하면서 핵심 기능을 동작 가능한 수준으로 구현하도록 유도한다.

## 구현 방향 요약

- 대상 사용자: 개발자 취업 준비생
- 제품 형태: 한국어 웹앱
- 제품 비전: AI 올인원 취업 준비 플랫폼
- 초기 핵심 기능: 자기소개 생성, 이력서 첨삭, 기업/직무 매칭, 피드백 리포트, 텍스트 기반 면접 연습
- 향후 확장: 음성/영상 면접, 표정/음성 피드백, 회사별 면접 준비, 모바일 앱
- 기술 방향: React 프론트엔드, Node.js 서버 런타임, 서버사이드 Gemini API 호출
- 데이터 방향: Firebase Auth/Firestore는 필요 시 사용, 첫 버전은 로컬 상태 기반으로 시작해도 됨

## 첫 프롬프트

```text
개발자 취업 준비생을 위한 한국어 AI 올인원 취업 준비 웹앱 프로토타입을 만들어줘.

이 앱은 사용자가 자기소개를 만들고, 이력서를 첨삭받고, 본인에게 맞는 기업/직무 방향을 추천받고, 면접 연습을 할 수 있는 서비스야. 장기적으로는 음성/영상 기반 실전 면접과 표정/음성 피드백까지 확장할 예정이지만, 첫 프로토타입에서는 텍스트 기반으로 핵심 흐름을 완성해줘.

사용자 문제:
- 자기소개 작성이 어렵다.
- 이력서가 경쟁력 있는지 모른다.
- 어떤 기업이나 개발 직무에 지원해야 하는지 모른다.
- 면접 연습을 할 환경이 부족하다.
- 프로젝트와 포트폴리오 경험을 채용 관점으로 표현하기 어렵다.

핵심 기능:
1. 대시보드
   - 자기소개 만들기, 이력서 첨삭, 기업/직무 매칭, 면접 연습, 종합 리포트로 이동할 수 있게 해줘.
   - 취업 준비 진행 상태를 한눈에 볼 수 있게 해줘.

2. 사용자 자료 입력
   - 목표 직무, 기술 스택, 프로젝트 경험, 포트폴리오 링크, GitHub 링크, 기존 이력서, 채용공고, 희망 기업 조건을 입력할 수 있게 해줘.
   - 입력한 자료는 이후 AI 분석에 함께 사용되도록 상태로 관리해줘.

3. 자기소개 생성
   - 사용자의 목표 직무, 프로젝트 경험, 기술 스택을 바탕으로 한국어 자기소개를 생성해줘.
   - 30초 자기소개, 1분 자기소개, 서류용 자기소개 중 선택할 수 있게 해줘.
   - 사용자가 제공하지 않은 경력이나 성과를 임의로 만들지 말고, 근거가 부족한 내용은 확인 필요로 표시해줘.

4. 이력서 첨삭
   - 이력서 내용을 항목별로 분석해줘.
   - 개발자 채용 관점에서 프로젝트 설명, 기술 스택 표현, 성과 표현, 직무 적합성을 피드백해줘.
   - 수정 전/후 예시와 수정 이유를 함께 보여줘.

5. 기업/직무 매칭
   - 사용자의 기술 스택, 프로젝트 경험, 선호 조건을 바탕으로 적합한 개발 직무와 기업 유형을 추천해줘.
   - 추천 이유, 관련 경험, 부족한 준비, 다음 액션을 함께 보여줘.
   - 실제 기업 데이터가 없으면 특정 기업을 단정하지 말고 기업 유형과 직무 방향 중심으로 추천해줘.

6. 면접 연습
   - 목표 직무와 난이도를 선택하면 면접 질문을 생성해줘.
   - 사용자가 텍스트로 답변하면 답변 구조, 기술 설명력, 구체성, 개선 방향을 피드백해줘.
   - 화면에는 향후 음성/영상 면접과 표정/음성 피드백이 추가될 수 있음을 반영하되, 지금은 실제 녹음/녹화 기능은 구현하지 않아도 돼.

7. 종합 피드백 리포트
   - 자기소개, 이력서, 기업/직무 매칭, 면접 답변을 종합해 리포트를 만들어줘.
   - 강점, 약점, 직무 적합도, 추천 방향, 개선 우선순위, 다음 액션을 보여줘.

기술 요구사항:
- React 기반 웹앱으로 만들어줘.
- Gemini API 호출은 서버사이드 Node.js 런타임에서 처리해줘.
- API 키는 클라이언트 코드에 노출하지 말고 서버 secret 또는 환경변수로만 사용해줘.
- AI 응답은 가능하면 JSON Schema 기반 structured output 형태로 다뤄줘.
- 프론트엔드 컴포넌트 안에 긴 프롬프트를 하드코딩하지 말고, AI 호출 로직과 프롬프트/스키마를 분리해줘.
- 첫 프로토타입은 저장소 없이도 동작하도록 만들되, Firebase Auth와 Firestore를 붙이기 쉬운 구조로 만들어줘.

디자인 요구사항:
- 한국어 UX 카피를 사용해줘.
- 첫 화면은 랜딩페이지가 아니라 실제 대시보드여야 해.
- SaaS/업무 도구처럼 차분하고 실용적인 UI로 만들어줘.
- 문서 편집과 피드백을 나란히 볼 수 있게 해줘.
- 카드 안에 카드를 중첩하지 말고, 반복 항목에만 카드를 사용해줘.
- 모바일과 데스크톱 모두에서 텍스트가 겹치지 않게 반응형으로 구성해줘.

안전 요구사항:
- 사용자가 제공하지 않은 경력, 프로젝트, 수상, 정량 성과를 만들어내지 마.
- 추천과 피드백이 합격을 보장하는 것처럼 표현하지 마.
- 이력서, 포트폴리오, 면접 답변은 민감한 개인정보로 간주해줘.
- 향후 음성/영상 기능을 추가할 때는 명시적 권한 요청과 보관 정책이 필요하다는 점을 코드 구조와 UI 문구에 반영해줘.

원하는 결과:
- 동작 가능한 프로토타입
- 대시보드, 자료 입력, 자기소개, 이력서 첨삭, 기업/직무 매칭, 면접 연습, 리포트 화면
- 샘플 데이터로도 흐름을 테스트할 수 있는 상태
- 이후 Firebase 저장, 문서 업로드, 음성/영상 면접으로 확장하기 쉬운 구조
```

## 후속 프롬프트 1: Firebase 저장 추가

```text
현재 앱에 Firebase Authentication과 Firestore를 추가해줘.

- Google 로그인으로 사용자가 로그인할 수 있게 해줘.
- 사용자별로 프로필, 자기소개, 이력서, 매칭 결과, 면접 연습 결과, 피드백 리포트를 저장해줘.
- 로그인하지 않은 사용자는 샘플/임시 모드로 체험할 수 있게 해줘.
- Firestore 접근 로직은 UI 컴포넌트와 분리해줘.
```

## 후속 프롬프트 2: 문서 업로드 추가

```text
이력서와 포트폴리오 PDF 업로드 기능을 추가해줘.

- PDF 파일을 업로드하면 서버사이드에서 Gemini Document Understanding을 사용해 내용을 분석해줘.
- 추출 결과를 사용자 프로필, 프로젝트 경험, 기술 스택, 이력서 항목으로 구조화해줘.
- 원본 파일과 추출 데이터는 분리해서 다룰 수 있게 구조를 잡아줘.
- 비지원 파일 형식과 너무 큰 파일에 대한 오류 상태를 UI에 표시해줘.
```

## 후속 프롬프트 3: 면접 기능 확장 준비

```text
면접 연습 화면을 향후 음성/영상 면접으로 확장하기 쉽게 리팩터링해줘.

- 현재 텍스트 답변 모드를 유지해줘.
- 향후 audio, video 모드를 추가할 수 있도록 타입과 UI 구조를 분리해줘.
- 브라우저 microphone/camera 권한 요청이 필요한 위치를 명확히 표시해줘.
- Gemini Live API와 ephemeral token을 붙일 수 있도록 서버 API 경로를 준비해줘.
```

## 후속 프롬프트 4: 발표용 목차와 회고 화면 추가

```text
프로젝트 발표용 화면을 추가해줘.

- 프로젝트 개요
- 프로젝트 목표
- 사용자 문제와 해결 방안
- 아키텍처 구성도
- 프로토타입 소개
- 팀원 각자 회고

이 화면은 실제 서비스 화면과 분리된 발표 모드로 접근할 수 있게 해줘.
```

## 확인 기준

첫 생성 후 아래를 확인한다.

- 첫 화면이 실제 대시보드인지.
- 한국어 UX가 자연스러운지.
- 자기소개, 이력서, 매칭, 면접, 리포트 흐름이 모두 보이는지.
- Gemini API 키가 클라이언트에 노출되지 않는지.
- AI 출력이 구조화되어 UI에 안정적으로 표시되는지.
- 사용자 경험을 과장하거나 허위로 만들어내는 문구가 없는지.
- 향후 음성/영상 면접으로 확장 가능한 구조인지.
